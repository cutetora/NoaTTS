# 更新履歴 (Changelog)

## v1.2.0 — 2026-06-19

### 📦 ワンクリック配布（ポータブル版）
Python も CUDA も自分で入れずに、**展開／インストールするだけ**で使えるようになりました。

- **ポータブルビルド** `build_portable.bat` — 再配置可能な Python を同梱し、ZIP を生成。
  - **THIN（既定・約75MB）**: torch/CUDA は同梱せず、**初回起動時に GPU に合わせて自動DL**（CUDA 自動検出）。GitHub Releases にそのまま置けるサイズ。
  - **FULL**（`set FULL=1`）: torch も同梱（オフライン向け）。
- **インストーラ** `installer.iss`（Inno Setup）— `NoaTTS-Setup.exe` を生成。`{localappdata}` へ導入し管理者権限不要。`set INNO=1 & build_portable.bat` で自動コンパイル。
- **初回セットアップ** — torch（CUDA自動検出）→ 依存 → **TTSモデル（Irodori-500M）まで先行ダウンロード**。2回目以降の起動が一瞬に。失敗時は原因別に案内し再実行可能。
- **入口を一本化** — ユーザーが触る入口は `NoaTTS.exe` のみ。補助 bat は `scripts\` に格納し、取り違えを防止。
- **リリース補助** `release.bat <tag>` — ビルド → 実機確認の確認 → 承認で GitHub Release 作成・成果物添付までを半自動化。
- 軽量モード(int4)の依存は `requirements-lite.txt` に分離（使う人だけ導入）。未導入環境では通常モデルへ自動フォールバックし、起動エラーを防止。

> 必要なのは **NVIDIA GPU＋最新ドライバ**だけ。Python / CUDA Toolkit の手動導入は不要です。

## v1.1.0 — 2026-06-19

### 🆕 新機能
- **⚡軽量モード** — int4 量子化モデル（`kizuna-intelligence/Irodori-TTS-500M-v3-int4`）に切り替えて VRAM を節約。読み上げ設定ウィンドウ／Voice Studio のトグルで ON/OFF でき、モデル別の VRAM 目安も表示します。
- **UIエンジンのアイドル自動解放** — 軽量モード時、一定時間（既定 180 秒）操作が無ければ Voice Studio のモデルを自動退避し、次の生成で自動再ロードします。
- **トレイの起動／終了を対称化** — 「❌ アプリを閉じる」で daemon・Voice Studio・読み上げ設定ウィンドウをまとめて停止。起動時には daemon も自動で立ち上がります。

### ⚡ VRAM 最適化
従来は読み上げ中に 4〜5GB まで跳ね上がっていた VRAM を、**待機 約1.3GB／読み上げ中 約1.5GB** に削減しました。

- 感情指定（caption）での VoiceDesign モデル（約2GB）の常駐を廃止し、感情は **emoji 駆動**（文中の 😭😠🥺 など）に統一。
- 読み上げ完了後・**1文ごと**に GPU 作業バッファを解放（`reserved` のせり上がりを防止）。
- テキストを**常に句点で分割**し、長文を1チャンクで合成しないことでピークを抑制。
- モデル切替時の VRAM リークを修正（`unload` 時の `gc.collect`）。

### 🛠 改善
- マスコット「ノア」の案内（チュートリアル・ヘルプ・FAQ検索）に軽量モードの説明を追加。
- 読み上げ設定ウィンドウの「使用モデル」が現在のモデルを正しく選択表示するように。
