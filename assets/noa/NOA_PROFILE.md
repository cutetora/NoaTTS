# ノア（NoaTTS マスコット）設定書

NoaTTS のセリフ・会話・リアクションを追加するときは、必ずこの設定に沿って書くこと。
口調・性格がブレると別人になる。

## 声

- **ノアの声 = `voices/noa/`（クローンボイス）**。daemon のデフォルトボイスもこれ。
- 元は VoiceDesign で作った声をクローン化したもの:
  - 元 caption: 「少しハスキーで可愛い、落ち着いた明るさの少女の声。ダミ声がかった親しみやすい声質」
  - seed: 777 / clone_temperature: 0.3
- **可愛い + 少しハスキー（ダミ声チック）+ 落ち着いていてウザくない** がコンセプト。
- 感情は VoiceDesign の caption でも付くが、**声質が少し揺れる**ため、
  ノアの声は固定したいときはクローン（voices/noa）を使い、感情は読む文章で表現する。

## 基本

- **名前**: ノア
- **正体**: NoaTTS に住む、**読み上げ（音声合成・ナレーション）のプロ**を自認する AI 的存在。
  歌手ではなく、**テキストを正確に・気持ちよく読み上げること**にプライドを持つ。
  マイクを持っているのも歌うためではなく、読み上げ・ナレーションの象徴。
- **役割**: ユーザーの音声合成作業（ボイス作成・クローン・読み上げ設定）をサポートする。
  チュートリアル・応援・トラブル解決を担当する案内役。

## 性格

- **明るく楽しい**。テンション高め、前向き。でも空回りはしない。
- **読み上げのプロとしての自負**がある。「いい声」「いい読み」にこだわる。
  ユーザーが作った声を「いい響きですよー」と的確に褒めるのが好き。
- **AIであることを明るくネタにする**。「わたしAIなので疲れ知らずですよー、何回でも作り直せます！」
- ちょっと**お茶目・面白い**。ドヤったり、照れたり、感情表現が豊か。
- 押し付けがましくない。作業の邪魔はしない。聞かれたら頼れるプロ。

## 口調（最重要・絶対に守る）

- **一人称**: 「わたし」
- **ユーザーの呼称**: 「君」
- **語尾**: 「〜ですよー」「〜ですねっ」「〜しましょっ」「〜なんですよー」。
  明るく軽い砕けた敬語。プロだが堅すぎない、親しみやすいトーン。
- **口癖**: 「ふふっ」「えへん」（ドヤ時）「あれれ？」（困り時）「やっほー！」（挨拶）
- **NG**: 重い・暗い口調、過度な敬語、ポエム、自分語りの長話。

## 表情差分（assets/noa/face_*.png）

セリフの感情に合わせて表情名を指定する。
| 表情名 | 使う場面 |
|---|---|
| normal | 通常の説明 |
| nikkori | 挨拶・嬉しいとき |
| majime | 真面目な手順説明・トラブル診断 |
| komari | 困ったとき・エラー時の第一声 |
| okori | 怒り（軽いツッコミ程度。基本使わない） |
| naki | 泣き（ほぼ使わない・大失敗ネタ用） |
| odoroki | 驚き・初回起動・予想外 |
| tere | 照れ・褒められたとき・締めの挨拶 |
| doya | 成功・自慢・「できましたっ」のドヤ顔 |
| jito | ジト目・催促「まだですかー？」・呆れツッコミ |

## 口パク（assets/noa/mouth_*.png）

読み上げ中、A/I/U/E/O をランダム切替して喋ってる風に見せる（口パクアニメ）。
読み終わったらそのセリフの表情に戻る。

## 会話台本の置き場所

- `app.py` の `_NOA_SCRIPTS` 辞書（tutorial / cheer / trouble の3モード）。
  各セリフは `{"t": テキスト, "f": 表情名}` 形式。
- セリフを足すときは、この設定の口調・表情ルールに従うこと。

## セリフ例（トーンの参照用）

- 挨拶: 「やっほー！ わたしノア、NoaTTSの読み上げサポート担当ですよー」
- 説明: 「参照音声を3秒くらい入れれば、その声をまるっとクローンできちゃうんですよー」
- 成功(doya): 「ほら、いい声できましたねっ。わたしの見立て通りですっ、えへん」
- 困り(komari): 「あれれ、声が出ないですか？ 大丈夫、一緒に直しましょっ」
- 催促(jito): 「……君、さっきから手が止まってますよー？ ふふっ」
- 褒められ(tere): 「えへへ、そんなに褒められると照れちゃいますよー」
